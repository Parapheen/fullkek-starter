# Example Endpoints and Integration Guide

This guide summarizes the baked-in endpoints that support the HTMX and Fixi.js demos generated by the starter.

## Overview

The scaffolder now includes:

1. **Tailwind, DaisyUI standalone, and Tailwind + Basecoat landing pages** with embedded HTMX and Fixi.js interactions
2. **Example router templates** for the standard library and Chi frameworks
3. **Conditional rendering** so only the selected frontend's endpoints are scaffolded

## File Structure

```
internal/templates/
├── features/
│   ├── frontend/
│   │   └── htmx/internal/transport/http/router-htmx.go.tmpl
│   ├── http/
│   │   ├── chi/internal/transport/http/router-examples.go.tmpl
│   │   └── standard/internal/transport/http/router-examples.go.tmpl
│   └── styling/
│       ├── tailwind/web/templates/pages/index.html.tmpl
│       ├── tailwind_basecoat/web/templates/pages/index.html.tmpl
│       └── daisyui/web/templates/pages/index.html.tmpl
```

## API Endpoints by Frontend Technology

### HTMX Endpoints

When HTMX is selected, the following endpoints are available:

#### `GET /api/counter`

Returns an HTML fragment with a Tailwind-styled counter card. The counter value is tracked in-memory when you run `make go`.

```html
<div class="rounded-2xl border border-slate-200 bg-white p-5 shadow-sm space-y-4">
  <header class="space-y-1">
    <h3 class="text-base font-semibold text-slate-900">Server counter</h3>
    <p class="text-sm text-slate-600">Rendered via HTMX.</p>
  </header>
  <div>
    <p class="text-sm text-slate-600">Current value</p>
    <span id="count" class="text-3xl font-semibold text-slate-900">0</span>
  </div>
  <button class="inline-flex items-center gap-2 rounded-lg bg-sky-600 px-4 py-2 text-white font-medium hover:bg-sky-700 transition"
          hx-post="/api/increment"
          hx-target="#count"
          hx-swap="innerHTML">
    Increment
  </button>
</div>
```

#### `POST /api/increment`

Returns the incremented counter value as plain text.

```html
1
```

### Fixi.js Endpoints

When Fixi.js is selected, the following endpoints are available:

#### `GET /api/counter`

Returns an HTML fragment for Fixi.js.

```html
<div class="rounded-2xl border border-slate-200 bg-white p-5 shadow-sm space-y-2">
  <h3 class="text-base font-semibold text-slate-900">Fixi.js fetched content</h3>
  <p class="text-sm text-slate-600">Counter: <span class="font-semibold text-slate-900">42</span></p>
</div>
```

#### `GET /api/todos`

Returns an HTML list of todos.

```html
<div class="rounded-2xl border border-slate-200 bg-white p-5 shadow-sm space-y-2">
  <h3 class="text-base font-semibold text-slate-900">Server todos</h3>
  <ul class="space-y-2">
    <li class="rounded-lg border border-slate-200 bg-slate-50 px-3 py-2 text-sm text-slate-700">Learn Fixi.js patterns</li>
    <li class="rounded-lg border border-slate-200 bg-slate-50 px-3 py-2 text-sm text-slate-700">Build reactive components</li>
    <li class="rounded-lg border border-slate-200 bg-slate-50 px-3 py-2 text-sm text-slate-700">Deploy your app</li>
  </ul>
</div>
```

#### `GET /fragments/toast/success`

When the Tailwind + Basecoat or DaisyUI styling options are paired with HTMX, this endpoint returns a
fragment that matches the selected component library. The generated page appends the toast to
`#toaster`, making it easy to replicate the pattern for your own notifications.
